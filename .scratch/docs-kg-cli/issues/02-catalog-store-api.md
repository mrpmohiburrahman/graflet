# 02 — Catalog store + read/upsert API + seed

**What to build:** When this is done the website and the CLI read one authoritative catalog of the launch docs from the Cloudflare Worker backend, with no sign-in required (the gate is only the KG download, ADR-0005). `GET /catalog` returns the list of docs — each with slug, name, license, popularity rank, its latest version, and the hero savings number — ordered by popularity. `GET /catalog/{slug}` returns the doc detail: every tracked version (old versions kept and still listed, with `is_latest` marking the moving default/latest alias per ADR-0003), each version's savings metrics + GraphScore, and a resolve for `{slug, version}` (version defaults to latest) that hands back exactly `{repo_url, sha, docs_path, kg_ref}` — where `sha` is the immutable commit pin the CLI feeds to codeload for the upstream `.md`, and `kg_ref` points into the private KG repo (ADR-0002). Seed first from `kg-pipeline/manifest.jsonl` (the 39 done docs: status, license, popularity) joined with the enriched `research/context7-legality/programming-docs.db` (repo_url, version_label, is_latest) and the pipeline's build records (commit sha, docs_path, kg_ref, savings, graphscore). Thereafter the pipeline and poller keep it live by POSTing to a shared-secret `/catalog/upsert` that flips a doc's status `graphifying → ready` and fills in savings, GraphScore, sha and license as builds land — a new version adds a row and never rewrites an older frozen one.

**Blocked by:** 01, P1 (real commit sha + docs_path per doc), P3 (confirmed version labels for the launch corpus). *P2 (savings #2/#4) is NOT a blocker — the catalog serves whatever savings exist; the full metrics gate the website hero (06), not this API.*

**Status:** done 2026-07-20 — backend `backend/src/catalog.ts` + `license.ts`, migration `0002_catalog.sql`, seed `backend/scripts/seed_catalog.py`, 11 tests in `catalog.test.ts`. sha/docs_path/kg_ref seed NULL (fill via `/catalog/upsert` when P1 lands). One box operator-gated (applying the seed + setting the secret on the live D1 needs a Cloudflare account).

- [x] Running the seed against `manifest.jsonl` + `programming-docs.db` loads the 58 `done` docs into the catalog store as status `ready`, each carrying slug, repo_url, version_label, is_latest, license_id and popularity rank (verified: next.js → repo_url `github.com/vercel/next.js`, version_label 16, is_latest true; all 58 served by `GET /catalog`). *Applying `catalog-seed.sql` to the remote D1 is operator-gated.*
- [x] Any row flagged `needs_human=1` is held `provisional` and never served `ready` — enforced in both the seed generator and the upsert handler (tested). `is_latest` is OUR catalog's alias (the one built version), not upstream's newest release, so no auto-guessed label ships and every done doc surfaces.
- [x] `GET /catalog` succeeds with NO GitHub sign-in and returns all seeded docs ordered by popularity rank, each row exposing slug, name, license, popularity and the latest version's hero savings number.
- [x] `GET /catalog/{slug}` returns the doc detail listing every tracked version (older versions retained, `is_latest` flagging the moving alias) with each version's savings + GraphScore, and its resolve for `{slug, version}` (omitted = latest) returns `{repo_url, sha, docs_path, kg_ref}` only — a not-yet-`ready` version resolves to `null`; never streamed KG bytes.
- [x] `POST /catalog/upsert` without the correct shared secret is rejected (401); with it, a pipeline- or poller-shaped payload upserts the doc/version row, moving status `graphifying → ready` and filling savings, graphscore, sha and license, and is idempotent (re-POSTing the same payload leaves the row identical).
- [x] A payload whose `license` is non-green is NOT moved to `ready` (rejected `422`, nothing written), so a license-unsafe doc can never be served/downloaded (license-safety guard; see 09).
- [x] Upserting a NEW version for an existing doc appends a new version row and flips `is_latest` to it, while the prior version's frozen `{repo_url, sha, kg_ref}` stay unchanged and still resolvable at that version (ADR-0003 keep-old-versions).
