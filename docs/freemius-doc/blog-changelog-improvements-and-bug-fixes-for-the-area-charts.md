[Changelog](https://freemius.com/changelog/) / Improvements and bug fixes for the area charts

Not so long ago, we introduced [general UX improvements](blog-changelog-new-geolocation-based-payments-distribution-chart-audience-analytics-charts-ux-improvements.md) to the analytics charts available in the Freemius Developer Dashboard. We noticed the “Area Chart” rendering was not up to the mark, especially if different categories had closer values. It also had a bug where the minimum value of the y-axis was wrongly calculated and there was no way to reset it.

[![Better area chart Freemius Developer Dashboard](https://freemius.com/blog/wp-content/uploads/2024/06/better-area-chart-freemius-developer-dashboard-1024x574.png)](https://freemius.com/blog/wp-content/uploads/2024/06/better-area-chart-freemius-developer-dashboard.png)

In this deployment, we fixed the bug and improved the default rendering.

[![Customizing area chart for setting y-axis in Freemius Developer Dashboard](https://freemius.com/blog/wp-content/uploads/2024/06/full-size-y-axis-area-chart-freemius-developer-dashboard-1024x761.png)](https://freemius.com/blog/wp-content/uploads/2024/06/full-size-y-axis-area-chart-freemius-developer-dashboard.png)

You will also get the option to set “Full Size” Y-Axis by clicking the “Customize” button.
