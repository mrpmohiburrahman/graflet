[Changelog](https://freemius.com/changelog/) / Checkout Layout Bug Fix for WordPress SDK 2.13

With the release of the [WP SDK 2.13](blog-changelog-freemius-wordpress-sdk-v2-13-0-released.md), we introduced an option to customize the in-dashboard Checkout experience. Soon after, we learned that setting the `layout` to `vertical` caused a visual glitch that affected the background of the checkout form.

[![Freemius Checkout in vertical layout inside the WordPress Admin](https://freemius.com/blog/wp-content/uploads/2025/11/fs-checkout-vertical-mode-wp-dashboard-1024x807.png)](https://freemius.com/blog/wp-content/uploads/2025/11/fs-checkout-vertical-mode-wp-dashboard.png)

With this week’s deployment, the issue has been fully resolved. No changes are needed on your end as the fix is applied directly via our Checkout application.
