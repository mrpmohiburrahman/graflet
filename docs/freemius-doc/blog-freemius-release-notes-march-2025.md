#### How to Install Freemius on Mobile and Desktop

![how to install Freemius on mobile and desktop](https://freemius.com/blog/wp-content/uploads/2025/03/how-to-install-freemius-on-mobile-and-desktop.jpeg)

📱 On Android

1. Open the **Freemius Developer Dashboard** in Chrome
2. Tap the **Profile menu** in the top right
3. Click the **Install App** button

🍏 On iOS

1. Open the **Freemius Developer Dashboard** in Safari
2. Tap the **Share** button
3. Scroll down and select **Add to Home Screen**

**![iOS freemius mobile installable version](https://freemius.com/blog/wp-content/uploads/2025/03/ios-freemius-mobile-installable-version.png)**

💻 On Desktop (Chrome)

1. Open the **Freemius Developer Dashboard**
2. Click the **install icon** in the address bar

Once installed, Freemius runs like a standalone app, making it easy to check sales, manage licenses, and track subscriptions from anywhere.

## Custom License Unit Labels: More Flexibility for SaaS & Applications

Freemius licensing has traditionally been based on site activations, which works well for WordPress plugins and themes. But software makers outside the WP ecosystem — those selling SaaS, desktop apps, or API-based services — need more flexibility.

![freemius custom license unit labels for saas and apps](https://freemius.com/blog/wp-content/uploads/2025/03/freemius-custom-license-unit-labels-for-saas-and-apps.jpeg)

![freemius custom license unit labels activations](https://freemius.com/blog/wp-content/uploads/2025/03/freemius-custom-license-unit-labels-activations.jpeg)

With [Custom License Unit Labels](blog-changelog-custom-license-unit-label-for-saas-and-applications.md), makers can now define the unit that best fits their product, ensuring a clearer and more intuitive buying experience.

- No more being locked into “site activations.” Set a custom label in product settings to reflect how your licenses are sold
- Works for all software types, including SaaS, desktop apps, Chrome extensions, and more
- Seamlessly integrated across checkout, email receipts, and customer portals, so customers always see the correct unit when purchasing a bulk license

**Example:** A SaaS platform for teams like Asana might use “Seats” instead of site activations. An API-based service could display “API Calls” or “Credits” instead.

With this update, the choice is yours.

**Important notes:**

- For applications that require license activation, the custom unit label will always be included in emails
- For SaaS products, the label will only appear in emails if you choose to expose the license key to users

## Incremental Updates: The Smart Way to Release Software

Users skip versions. They delay updates. They wait until a new feature or a security fix forces their hand.

But what happens when a skipped version contains critical code that must run before moving forward?

Without a way to enforce updates in sequence, a user might jump straight from version 1.2 to 1.6, skipping over crucial logic introduced in 1.4. If that missing step was essential — like a database migration or an API transition — the software could break.

Freemius’ [new incremental updates system](blog-changelog-new-feature-incremental-updates-for-your-wordpress-plugins-themes.md) gives makers full control over the update path. By anchoring a version, makers can require users to install it before moving to the newest release, ensuring everything runs as intended.

- No more breakages from skipped updates. Critical scripts and logic execute in the right order
- Fewer support headaches. Updates happen smoothly without unexpected failures
- A first in the WordPress space. No other platform offers this level of update control

![freemius incremental release feature](https://freemius.com/blog/wp-content/uploads/2025/03/freemius-incremental-release-feature.jpeg)

![freemius incremental release feature](https://freemius.com/blog/wp-content/uploads/2025/03/freemius-incremental-release-feature-1.jpeg)

With a simple toggle, makers trigger a powerful safeguard that keeps software stable, structured, and secure as it evolves.

## New Webhook: Get Notified When Affiliates Are Ready for Payouts

Affiliate payouts, now with real-time notifications. We’ve introduced [a webhook event](blog-changelog-new-webhook-get-notified-when-its-time-to-pay-affiliates.md) that fires automatically when an affiliate reaches the $100 payout threshold.

![affiliate payout freemius webhook](https://freemius.com/blog/wp-content/uploads/2025/03/affiliate-payout-freemius-webhook.jpeg)

Previously, Freemius sent monthly email notifications listing affiliates who qualified for payouts. With this webhook, makers can fully automate the process, eliminating the need to manually track emails or process payments.

Once a month, as our system calculates pending affiliate payouts, it will trigger the `affiliate.payout.pending` webhook event for all affiliates eligible for a payout after crossing the $100 threshold.

Since PayPal account details are already available, payments can be triggered programmatically via PayPal’s API, reducing administrative overhead and ensuring affiliates receive their earnings without delays.

![affiliate payout freemius webhook](https://freemius.com/blog/wp-content/uploads/2025/03/affiliate-payout-freemius-webhook-1.jpeg)

This is an important step toward full affiliate payout automation, which is already on our roadmap. We know this is a highly requested feature, and while it will take time to implement, we appreciate your patience and remain committed to making it happen.

For guidance on setting up and managing webhooks, check out the [Events & Webhooks documentation](help-documentation-marketing-automation-events-webhooks.md).
