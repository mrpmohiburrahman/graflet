[Changelog](https://freemius.com/changelog/) / One-Click Customer Portal Login via API

This week we’re releasing a new capability that lets you generate auto-login links to the [Customer Portal](help-documentation-users-account-management.md) for your buyers. We understand that in today’s SaaS & App marketplace, development speed is critical, so we aim to offload as much infrastructure and UX complexity from you as possible. As part of that, Freemius already provides a hosted [Customer Portal](help-documentation-users-account-management.md) where buyers can manage subscriptions, purchases, and even join the [affiliate program](help-documentation-users-account-management-earn-becoming-an-affiliate.md).

Until now, there was no way to generate login links that automatically authenticated customers. Buyers always had to enter their Freemius password to access the portal. With today’s rollout, this is no longer required. You can now make an [API call](https://docs.freemius.com/api/products/generate-portal-login-link) to generate a secure magic link and redirect your buyers directly into the Customer Portal without any manual login step.

### Generating the Magic Login Link

You need to make an [API call](https://docs.freemius.com/api/products/generate-portal-login-link) to the following endpoint using [Bearer Token authentication](https://docs.freemius.com/api/section/bearer-token-auth):

```
POST https://api.freemius.com/v1/products/{product_id}/portal/login.json
{"email": "[email protected]"}
```

The response will be a JSON object with the following properties:

- `link` – The magic login link
- `token` – A login token for manual or more advanced integrations

[![Portal login API call](https://freemius.com/blog/wp-content/uploads/2026/02/portal-login-link-api-call-email-1024x631.png)](https://freemius.com/blog/wp-content/uploads/2026/02/portal-login-link-api-call-email.png)

Once generated, you should immediately redirect your customer to the `link`, and they will be automatically logged in to the Customer Portal.

[![Logged in Customer Portal](https://freemius.com/blog/wp-content/uploads/2026/02/portal-logged-in-1024x591.png)](https://freemius.com/blog/wp-content/uploads/2026/02/portal-logged-in.png)

More detailed documentation will follow shortly. Until then, please note that you can generate the login link using either:

1. `email` – If you don’t store the Freemius [User ID](https://docs.freemius.com/api/users/retrieve)
2. `id` – If you do store it, which is the preferred approach

Also note that the link generated by the API call is short-lived and expires after 5 minutes. You should only generate it when the customer explicitly requests access to the portal, and then redirect them immediately.
