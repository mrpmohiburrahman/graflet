[Changelog](https://freemius.com/changelog/) / Checkout-related enhancements for the Developer Dashboard

We’ve rolled out a few small but important bug fixes and improvements to the Developer Dashboard, focusing on various checkout-related actions.

### Fixed checkout URL for SaaS products under pricing

[![](https://freemius.com/blog/wp-content/uploads/2025/02/developer-dashboard-checkout-link-1024x347.png)](https://freemius.com/blog/wp-content/uploads/2025/02/developer-dashboard-checkout-link.png)

We identified a regression where clicking the **“Checkout Link”** button did not generate the correct URL for [SaaS products](help-documentation-selling-with-freemius-saas-integration.md) on the [**Plans** page](help-documentation-getting-started-setup-product-pricing-plans-refunds.md). This issue has now been fixed.

### New license action to copy payment method update link

You will now find a new option under the “Action” menu in Licenses to copy the payment method update link. This is particularly useful for customer support when you need to quickly share a payment update link—for example, when a customer’s credit card is expiring or their PayPal account is invalid.

[![](https://freemius.com/blog/wp-content/uploads/2025/02/dashboard-payment-method-update-link.png)](https://freemius.com/blog/wp-content/uploads/2025/02/dashboard-payment-method-update-link.png)

Please note that this link is provided for your convenience only. Freemius still handles [dunning](help-documentation-selling-with-freemius-license-renewals-mechanism.md) and automatically sends payment-related emails for both card and PayPal issues. Additionally, buyers can update their payment method directly through the [Customer Portal](help-documentation-users-account-management.md) at any time.
