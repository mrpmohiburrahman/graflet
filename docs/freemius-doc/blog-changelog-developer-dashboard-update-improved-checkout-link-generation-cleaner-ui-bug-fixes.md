[Changelog](https://freemius.com/changelog/) / Developer Dashboard Update – Improved Checkout Link Generation, Cleaner UI & Bug Fixes

This week we’re rolling out several meaningful UI/UX improvements and a few bug fixes to the Developer Dashboard.

### Smarter Get Checkout Button

We’ve reorganized the links under the “Get Checkout” button on the [Plans](help-documentation-saas-saas-plans-pricing.md) page, making it much easier to grab your [no-code checkout links](https://freemius.com/help/documentation/checkout/hosted-checkout/).

[![Get Checkout button UX improvement](https://freemius.com/blog/wp-content/uploads/2025/12/plan-page-no-code-checkout-link-1024x593.png)](https://freemius.com/blog/wp-content/uploads/2025/12/plan-page-no-code-checkout-link.png)

Sandbox and production links are now grouped together and include a dedicated copy button for convenience. You can still click them to open in a new tab, just like before.

### JS-Based Sandbox Example

Under the same “Get Checkout” menu, selecting “Overlay Code” now shows a JS-based sandbox example powered by our [official JS SDK](help-documentation-saas-sdk-js-sdk.md).

[![Checkout JS based sandbox example](https://freemius.com/blog/wp-content/uploads/2025/12/checkout-overlaycode-js-sandbox-1024x719.png)](https://freemius.com/blog/wp-content/uploads/2025/12/checkout-overlaycode-js-sandbox.png)

This helps [SaaS makers](help-documentation-saas.md) quickly spin up a local integration environment without digging through documentation.

### Enhanced Empty State for the Plans Page

The Plans page now has a more thoughtful empty state when no plans exist.

[![Empty plan page](https://freemius.com/blog/wp-content/uploads/2025/12/empty-plan-page-1024x566.png)](https://freemius.com/blog/wp-content/uploads/2025/12/empty-plan-page.png)

Instead of a plain “Add Plan” button, we now show a clearer explanation of what a plan is and why you might create one, along with a link to helpful documentation for first-time makers.

### Other Notable Fixes

- Added a tooltip explaining what the Personal Success Manager [support feature](help-documentation-wordpress-setup-product-pricing-plans-refunds.md#configure-a-plans-support-channels) covers for makers.
- Cleaned up the license table layout for SaaS products.
- Fixed a small UI glitch in the sidebar [onboarding video](blog-changelog-developer-dashboard-gets-ui-fixes-and-a-new-video-walkthrough-for-easier-onboarding.md#improved_onboarding_experience) and improved the overall responsiveness.
- Resolved an edge case where a plan wasn’t being saved for newly created products.
- Fixed a visual glitch on the “[Emails](help-documentation-marketing-automation-email-settings.md)” page under Store.
- Improved the confirmation dialog when deleting a version from the [deployment section](help-documentation-release-management-deployment.md).
- Fixed layout overflow on the “[Cancel Trial](help-documentation-wordpress-free-trials.md)” button under “Site.”
