We have deployed two minor UI improvements to our Checkout this week.

### Fixed Padding Issue in Zip/Postal Code Input

[![](https://freemius.com/blog/wp-content/uploads/2025/03/checkout-zip-postal-code-1024x529.png)](https://freemius.com/blog/wp-content/uploads/2025/03/checkout-zip-postal-code.png)

As a regression to our newly introduced [UX improvements for the VAT Number UI](blog-changelog-checkout-ui-ux-improvements.md#:~:text=VAT%20Number%20Validation%20UX%20Improvement), we noticed there was extra left-over padding on the zip/postal code input. This has now been fixed.

### Removed Money-Back Guarantee UI from Free Plans

[![](https://freemius.com/blog/wp-content/uploads/2025/03/freemius-checkout-saas-free-plan-1024x682.png)](https://freemius.com/blog/wp-content/uploads/2025/03/freemius-checkout-saas-free-plan.png)

We noticed that we’ve been incorrectly showing the money-back guarantee UI when the Checkout is opened in the context of a Free plan. The same has now been fixed.
