[Changelog](https://freemius.com/changelog/) / RTL Support Comes to Email Customization

This week, we’re rolling out a big update to our [Email Customization](help-documentation-marketing-automation-email-content-customization.md) feature.

So far, we have supported adding custom content to transactional emails, making it possible to include useful information such as onboarding instructions, access links, and more. However, the custom content section did not fully support RTL mode yet. With the release of [RTL support in our Checkout](blog-changelog-rtl-language-support-comes-to-freemius-checkout.md), it made sense for our emails to follow the same flexibility.

[![Email customization RTL support in Freemius](https://freemius.com/blog/wp-content/uploads/2026/05/email-customization-rtl-support-1024x585.png)](https://freemius.com/blog/wp-content/uploads/2026/05/email-customization-rtl-support.png)

With that in mind, today we are rolling out RTL support for Email Customization. Simply head to the [Email Customization section](help-documentation-marketing-automation-email-content-customization.md), where you can now seamlessly enter RTL text and the system will automatically detect the text direction.

If you want to enter mixed text or force a specific direction, you can also explicitly set the **Custom Text Direction** option.

In addition, there is now an option to change the position of the custom section.

[![Email customization section position](https://freemius.com/blog/wp-content/uploads/2026/05/email-customization-section-position-1024x691.png)](https://freemius.com/blog/wp-content/uploads/2026/05/email-customization-section-position.png)

This is useful if you want to add fully localized custom instructions before the original email content.

We hope you find this feature useful. If you have any feedback or notice any issues, please feel free to reach out to us through support by logging in to the [Developer Dashboard](https://dashboard.freemius.com).
