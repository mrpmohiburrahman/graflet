[Changelog](https://freemius.com/changelog/) / New feature: White-labeling a specific pricing in a plan

We’ve implemented yet another feature request from our partners. From now on, you can mark a particular pricing as White Labeled. After doing that, every new license created for that pricing will be marked as White Labeled by default.

[![White-labelling pricing on a plan Freemius Developer Dashboard](https://freemius.com/blog/wp-content/uploads/2023/08/white-labelling-pricing-scaled.jpg)](https://freemius.com/blog/wp-content/uploads/2023/08/white-labelling-pricing-scaled.jpg)

This means when someone activates the license on a site, the WP SDK will not show sensitive information on the Accounts page.

[![Regular vs White-Labeled accounts page](https://freemius.com/blog/wp-content/uploads/2023/08/regular-vs-white-labeled-accounts-page-scaled.jpg)](https://freemius.com/blog/wp-content/uploads/2023/08/regular-vs-white-labeled-accounts-page-scaled.jpg)

This helps in situations where you create some sort of bulk pricing for an agency and the agency wants the license marked as White Labeled by default. Please read more about the white-labeling feature [here](blog-white-label-mode-url-whitelisting-user-dev-dashboard-upgrades-coupon-improvements.md#white_label_mode).
