[Changelog](https://freemius.com/changelog/) / Developer Dashboard Improvements: Templates, Higher Bulk Pricing & Webhooks

We have rolled out the following quality-of-life improvements to the [Developer Dashboard](https://dashboard.freemius.com/).

### Template Under WordPress Products

We realized in the [last release](blog-changelog-clearer-templates-kits-and-small-dashboard-improvements.md) we accidentally removed the “[Template](help-documentation-selling-templates.md)” type from the [WordPress](help-documentation-wordpress.md) category. Many makers use Freemius to sell WordPress-related templates (for example, page builder templates or customization templates for plugins), so this made it harder to locate the correct product type during setup.

[![Templates inside WordPress category](https://freemius.com/blog/wp-content/uploads/2026/02/new-product-form-with-template-1024x686.png)](https://freemius.com/blog/wp-content/uploads/2026/02/new-product-form-with-template.png)

To address this, the Template type has now been added back under the WordPress Products category in the new product form.

### Higher Bulk Pricing Support

Based on feedback from makers selling large credit bundles, we have increased the quota limit of [pricing](help-documentation-saas-saas-plans-pricing.md#plans-customization) to support values up to 100,000,000 (100 million).

[![Bulk pricing supporting 100 million](https://freemius.com/blog/wp-content/uploads/2026/02/freemius-bulk-pricing-1024x1022.png)](https://freemius.com/blog/wp-content/uploads/2026/02/freemius-bulk-pricing.png)

This makes it easier to configure plans for products that operate with high-volume credit systems.

### More Exposure to Webhooks

To improve discoverability, [webhooks](help-documentation-saas-events-webhooks.md) are now available as a top-level menu item instead of being nested under “Integration”.

[![Webhook as top level menu](https://freemius.com/blog/wp-content/uploads/2026/02/freemius-webhook-in-dashboard-1024x595.png)](https://freemius.com/blog/wp-content/uploads/2026/02/freemius-webhook-in-dashboard.png)

We received consistent feedback from SaaS & App makers that [webhooks are a core part of the integration](help-documentation-saas-saas-integration.md#creating-a-webhook-listener). Promoting it to a primary navigation item makes the integration workflow more direct and aligned with how modern SaaS products are built.
