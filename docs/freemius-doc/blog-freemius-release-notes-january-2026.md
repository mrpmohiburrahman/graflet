## Refund Policies for Usage-Based Pricing Now Support Unused Credits

As part of a broader documentation and localization effort, we reviewed our refund policies and found they no longer reflected how modern usage-based products are used in practice.

Our refund policies have [since been updated to allow refunds for unused credits only](blog-changelog-refund-policies-for-usage-based-pricing-now-support-unused-credits.md).

[![Update Refund Policy section in the Freemius dashboard](https://freemius.com/blog/wp-content/uploads/2026/02/update-refund-policy-section-in-the-freemius-dashboard.jpeg)](https://freemius.com/blog/wp-content/uploads/2026/02/update-refund-policy-section-in-the-freemius-dashboard.jpeg)

This helps prevent refund abuse while keeping policies fair and transparent for buyers, especially in cases where value is consumed immediately.

For example, if you’re running a generative AI product with subscriptions and credit top-ups, you may choose to offer a money-back guarantee only for unused credits.

As part of this update, we also:

- refreshed the [relevant documentation](help-documentation-selling-with-freemius-refund-policy.md#consumptive-usage)
- updated legal terms to cover usage-based consumption

This support is currently only available for SaaS-type products. If you have another product type with a valid usage-based use case, [our team can help explore enabling this option](https://freemius.com/cdn-cgi/l/email-protection#9be8eeebebf4e9efdbfde9fefef6f2eee8b5f8f4f6).

## Clearer, Faster Documentation for AI-Assisted Builds

More builders are using AI tools to design, ship, and iterate on their products, including a growing number of vibe coders building with AI-assisted workflows.

As part of an ongoing effort to improve clarity and usability, we’ve made targeted improvements to how Freemius documentation is structured and how it supports modern, AI-assisted builds.

### Filter Documentation by Product Type

[Documentation can now be filtered by product type](blog-changelog-filter-documentation-by-product-type-plus-improved-ai-llm-guidance.md#new_feature_filter_documentation_by_product_type), allowing you to focus on guides that are relevant to your setup.

You can select SaaS & Apps or WordPress in the documentation to:

- view a curated subset of content for that product type
- see pages adapt based on the selected category — for example, the [Sales Analytics documentation](help-documentation-analytics-sales.md) now reflects the chosen product type where applicable

[![Freemius Documentation page showing filter by product type](https://freemius.com/blog/wp-content/uploads/2026/02/freemius-documentation-page-showing-filter-by-product-type.jpeg)](https://freemius.com/blog/wp-content/uploads/2026/02/freemius-documentation-page-showing-filter-by-product-type.jpeg)

This update reduces noise and makes it easier to find the right guidance without digging through generic docs.

### Better AI and LLM Guidance

We recognize the growing importance of building with LLMs (large language models) and AI-powered tooling.

To make Freemius documentation easier to consume by AI scrapers, agents, and developer tools, we’re now [surfacing dedicated resources directly in the docs footer](blog-changelog-filter-documentation-by-product-type-plus-improved-ai-llm-guidance.md#more_guidance_for_llm_and_ai):

- Bundled documentation
- Summarized reference files
- Access to our [OpenAPI specification](https://docs.freemius.com/api)

![Freemius Docs footer with dedicated resources](https://freemius.com/blog/wp-content/uploads/2026/02/freemius-docs-footer-with-dedicated-resources.jpeg)

Documentation remains a never-ending investment for us. These improvements are part of a broader effort to keep the docs clear, current, and aligned with how software is built today.

## Customizable Proration Period for Lifetime License Upgrades

Previously, upgrading a lifetime license often meant paying the full difference, regardless of how long the original license had been in use. This understandably led to frustration, abandoned upgrades, or support requests.

To address this, we introduced proration for lifetime license upgrades — initially with a 7-day window, and later expanded to 30 days based on feedback. While this helped, it became clear that a one-size-fits-all proration period still didn’t work for everyone.

### Makers Now Control the Lifetime License Proration Window

[Proration periods for lifetime license upgrades are now](blog-changelog-customizable-proration-period-for-lifetime-license-upgrades.md) [fully customizable](blog-changelog-customizable-proration-period-for-lifetime-license-upgrades.md), putting the decision in the hands of the maker.

[![Freemius Checkout with Lifetime License Proration ](https://freemius.com/blog/wp-content/uploads/2026/02/freemius-checkout-with-lifetime-license-proration.jpeg)](https://freemius.com/blog/wp-content/uploads/2026/02/freemius-checkout-with-lifetime-license-proration.jpeg)

You can now choose what makes sense for your business and your customers, balancing fairness for buyers with your pricing strategy.

Makers can configure the proration period as:

- A fixed period (for example, a month or a year)
- A custom duration
- Unlimited

[![Prorated period config in the Freemius dashboard](https://freemius.com/blog/wp-content/uploads/2026/02/prorated-period-config-in-the-freemius-dashboard.jpeg)](https://freemius.com/blog/wp-content/uploads/2026/02/prorated-period-config-in-the-freemius-dashboard.jpeg)

This means buyers who upgrade after using a lifetime license for some time don’t necessarily pay the same price as someone upgrading immediately. Makers also no longer need to handle these situations manually with coupons or support tickets.
